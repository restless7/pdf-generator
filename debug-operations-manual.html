04A. Workforce Management

This section details the processes and tools for managing the ICE workforce, inspired by professional call center and sales operations.

Daily Operations Rhythm

Morning Briefing ("Huddle")

  • Frequency: Daily, at the start of each shift.
  • Duration: 10-15 minutes.
  • Led By: Supervisor.

Agenda: Review previous day's performance, announce today's targets, highlight urgent leads/escalations, address agent questions, motivational close.

End-of-Day Wrap-Up

Agents update lead statuses, log notes, and flag any end-of-day hand-offs. Supervisor reviews dashboards and prepares notes for next morning briefing.

Task Assignment Methodology

Supervisors use the Agent Control Panel (/portal/agents) to manage workload.

  1. Current Workload: Agents with lower active task counts receive new assignments first.
  2. Specialization Match: Leads are matched to agents specializing in the relevant program.
  3. Round Robin: For unspecialized leads, distribute evenly across available agents.
  4. Manual Override: Supervisors can manually re-assign leads based on judgment.

Quality Assurance (QA) Process

  1. Sampling: Supervisor reviews a random sample of 5-10% of agent interactions per month.
  2. Scoring: Each interaction is scored on Professionalism, Accuracy, Client Rapport, Closure (1-5 each).
  3. Feedback: Results shared individually. Trends discussed in weekly team meetings.
  4. Action Items: Agents scoring below threshold receive targeted coaching.

Performance Monitoring (Real-Time Dashboards)

MetricDescription
Agent StatusOnline, Offline, On Break. (Agent.isOnline, Agent.lastSeen)
Lead Response TimeTime from lead creation to first contact.
Conversion Rate% of leads converted per agent. (Lead.stage = 'CONVERTED')
Task CompletionOpen vs. completed tasks. (Task.status)
Daily ActivityCalls made, emails sent, leads contacted.

Lead Capture (ETL Process)

  1. Data Ingestion: Leads from external sources (spreadsheets, API webhooks) are ingested via ETL jobs.
  2. Data Normalization: Raw data is mapped to the Person and Lead models.
  3. Deduplication: System checks for existing Person by email or cedula.
  4. Creation: New Lead record created and linked to Person, Service, and Event (if applicable).

Lead Scoring

Leads are automatically scored (0-100) to prioritize follow-up.

Fit25%Program compatibility, age, country, eligibility.
Intent25%Program interest indicated, budget signals.
Engagement20%Email opens, replies, calls answered, events attended.

Recency10%Time since last contact (recent = higher).

Conversion Probability & Estimated Value

  • estimatedValue: conversionProbability * expectedDealValue

(No) -> Lost

Step 1: Client Accepts Offer

  • Agent presents program details, pricing, and terms.

09. Visa Rescheduling Operations

This section details the operational workflow for the ICE Visa Rescheduler Bot, the core proprietary product.

Overview

  1. Logs into US visa appointment portals using client credentials.
  2. Monitors for earlier available appointment slots.
  3. Automatically reschedules the appointment if a better slot is found.
  4. Notifies the client via Email/SMS.

Operational Workflow

Agent -> System: Register
System -> Queue: Create Job
Worker loop: Portal Login -> Check Dates -> Available?
If Yes: Reschedule -> Notify

Step 1: Applicant Registration

  1. Agent creates Person, Appointment, Preference, and Credential records.

Step 2: Job Creation

System creates a Job in the BullMQ queue (type: 'RESCHEDULE'). Job is scheduled to run at intervals defined by SearchRule.

Step 3: Automated Monitoring

The Worker service (Playwright) picks up job, launches headless browser, logs in, searches for appointments. Takes screenshot at each step for audit trail.

Step 4: Rescheduling

If an earlier slot is found: Worker completes the reschedule flow, receives new confirmation, creates RescheduleAttempt record, updates Appointment, triggers notification.

Step 5: Notifications

Clients are notified via multi-channel system: Email (Resend API or SMTP), SMS (Twilio).

Monitoring & Audit

  • Dashboard: /dashboard shows active applicants, queue depth, success rates.
  • Screenshots: Stored in screenshots/ directory.
  • Audit Log: All actions logged to AuditLog for compliance.

Error Handling & Escalation

Error TypeAction
Login FailureNotify agent to verify credentials. Mark job as failed.
Portal UnavailableRetry with exponential backoff. Alert Supervisor after N failures.
Captcha DetectedLog and pause. May require manual intervention.
Account LockedImmediately notify client and agent. Pause all jobs for that applicant.
Core Product Workflow

Agent Registers Applicant → System Creates Job → Worker Monitors Portal (Loop) → If Earlier Date Found: Reschedule & Notify → Log Attempt Result

  2. Enrollment.status = 'COMPLETED'.
